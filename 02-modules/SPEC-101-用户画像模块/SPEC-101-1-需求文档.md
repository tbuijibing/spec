# SPEC-101-1 需求文档

> 模块：用户画像模块 | 服务：`globehive-base:8300` | 版本：v1.0 | 日期：2026-03-29

---

## 1. 模块概述

### 1.1 模块定位

用户画像模块负责沉淀用户静态信息、扩展属性、社交账号、地址信息、家庭信息、标签与完整度指标，为会话消息、智能话术、主动营销、用户探索与商品匹配提供统一的用户认知基础。

### 1.2 业务背景

- 平台需要在多渠道沟通过程中形成统一用户认知
- 客服与 AI 回复需要基于画像做个性化表达与推荐
- 用户画像完整度直接影响探索策略和主动营销效果

### 1.3 模块边界

| 在范围内 | 在范围外 |
|----------|----------|
| 用户基础信息 CRUD | 用户登录/注册（BladeX 负责） |
| 标签管理（增删查） | AI 实时行为推断（SPEC-105） |
| 画像完整度计算 | 渠道绑定（SPEC-109） |
| ONE ID 合并 | 营销推送（SPEC-106） |
| Excel 批量导入/导出 | 财务账户（SPEC-116） |

---

## 2. 用户故事

### US-101-01 获取用户画像

**作为** 客服、运营或 AI 服务
**我希望** 获取指定用户的完整画像
**以便于** 在沟通前快速了解用户背景和偏好

**验收标准：**
- AC1：返回真实姓名、性别、生日、职业、收入范围等基础信息
- AC2：返回家庭成员、纪念日、收货地址、社交账号等扩展信息（JSON 字段）
- AC3：返回用户标签列表与画像完整度（0-100）
- AC4：按 `tenant_id` + `user_id` 双维度隔离查询

---

### US-101-02 更新用户画像

**作为** 客服或系统服务
**我希望** 对画像字段进行增量更新
**以便于** 在沟通过程中持续丰富用户认知

**验收标准：**
- AC1：支持单字段更新，不影响其他字段
- AC2：更新后自动重算完整度（0-100）
- AC3：写入审计日志（`ai_audit_log`，异步）
- AC4：并发更新时 Last-Write-Wins

---

### US-101-03 标签管理

**作为** 运营人员
**我希望** 为用户添加、删除、查询标签
**以便于** 进行精准分群和个性化运营

**验收标准：**
- AC1：标签以 JSON 数组存储，支持多标签
- AC2：同一标签不重复（自动去重）
- AC3：支持按关键词搜索用户（含标签搜索）

---

### US-101-04 ONE ID 合并

**作为** 系统管理员
**我希望** 将同一用户的多个渠道账号合并到主账号
**以便于** 建立跨渠道统一用户认知

**验收标准：**
- AC1：指定主账号 ID + 被合并账号 ID
- AC2：被合并账号的渠道绑定全量迁移至主账号
- AC3：被合并账号 `status=2`（已合并）
- AC4：整个操作在单一事务内完成
- AC5：操作写入审计日志

---

### US-101-05 批量导入

**作为** 运营人员
**我希望** 通过 Excel 批量导入用户画像
**以便于** 快速初始化历史用户数据

**验收标准：**
- AC1：可下载 Excel 模板（含字段说明）
- AC2：支持最多 1000 条/次批量导入
- AC3：重复 `user_id` 自动覆盖（Update）
- AC4：返回导入结果（成功 N 条，失败 M 条 + 错误行详情）

## 3. 业务规则

| 规则 | 说明 |
|------|------|
| 画像完整度 | 每个已填字段 +5 分，满分 100；基础字段权重 × 1，扩展字段权重 × 0.5 |
| 租户隔离 | 所有查询必须带 `tenant_id`，跨租户查询仅超级管理员可用 |
| 标签去重 | `addTags` 接口自动去重，不抛异常 |
| ONE ID 前置校验 | 主账号不能是已被合并账号（`status=2`） |

---

## 4. 非功能性需求

| 指标 | 要求 |
|------|------|
| 响应时间 | P99 < 500ms（单用户查询）|
| 吞吐量 | 支持 100 QPS 的画像查询 |
| 安全 | 跨租户访问 403，SQL 注入防御 |
| 可用性 | 99.9% SLA（依赖 `globehive-base` 服务可用性） |
