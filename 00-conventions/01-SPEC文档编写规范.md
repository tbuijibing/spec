# SPEC 文档编写规范

> 适用范围：所有 `SPEC-XXX-模块名/` 目录下的文档
> 参考示例：[`../SPEC-101-用户画像模块/`](../SPEC-101-用户画像模块/)

---

## 一、9 类文档总览

每个 SPEC 模块必须包含以下 9 类文档，编号连续，命名格式 `SPEC-{N}-{序号}-{类型}.md`：

| 序号 | 类型 | 文件名 | 核心内容 | 编写时机 |
|------|------|--------|----------|----------|
| 1 | 需求文档 | `SPEC-XXX-1-需求文档.md` | 用户故事、验收标准、约束 | 开发前 |
| 2 | 技术设计文档 | `SPEC-XXX-2-技术设计文档.md` | 架构、分层、API 设计、关键算法 | 开发前 |
| 3 | UI/UX 设计文档 | `SPEC-XXX-3-UI-UX设计文档.md` | 页面结构、交互规范、组件规格 | 开发前 |
| 4 | 任务文档 | `SPEC-XXX-4-任务文档.md` | 阶段任务拆解、状态跟踪、DoD | 开发中 |
| 5 | 验收文档 | `SPEC-XXX-5-验收文档.md` | 全量测试规范（接口/功能/安全/性能）| 开发前 |
| 6 | 数据库文档 | `SPEC-XXX-6-数据库文档.md` | DDL、索引、字段说明、ER 图 | 开发前 |
| 7 | 快速排查手册 | `SPEC-XXX-7-快速排查手册.md` | 常见错误、排查步骤、解决方案 | 开发中/后 |
| 8 | 可追溯性矩阵 | `SPEC-XXX-8-可追溯性矩阵.md` | 需求↔API↔代码↔测试映射 | 开发完成后 |
| 9 | 验收报告 | `SPEC-XXX-9-验收报告.md` | 验收结论、代码清单、测试结果 | 验收阶段 |

## 二、各文档格式规范

### 2.1 需求文档（-1）

**必含章节：**
```
# SPEC-XXX-1 需求文档
## 1. 模块概述（定位、背景、边界）
## 2. 用户故事（US-XXX-NN 格式，含验收标准）
## 3. 业务规则与约束
## 4. 非功能性需求（性能、安全、可用性）
## 5. 版本历史
```

**用户故事格式：**
```markdown
### US-XXX-01 {故事标题}
**作为** {角色}
**我希望** {功能}
**以便于** {目标}

**验收标准：**
- AC1: ...
- AC2: ...
```

---

### 2.2 技术设计文档（-2）

**必含章节：**
```
# SPEC-XXX-2 技术设计文档
## 1. 架构设计（分层架构图 + 职责说明）
## 2. 数据流设计（关键业务流程序列图）
## 3. API 接口设计（端点、请求/响应格式）
## 4. 关键实现（算法、缓存策略、事务设计）
## 5. 依赖关系（服务间依赖、外部依赖）
## 6. 版本历史
```

---

### 2.3 UI/UX 设计文档（-3）

**必含章节：**
```
# SPEC-XXX-3 UI/UX 设计文档
## 1. 页面概述（路由、组件路径、设计文件位置）
## 2. 页面结构（布局树形图）
## 3. 各页面/组件规格（字段、交互、状态）
## 4. 设计规范引用（颜色、间距、组件库）
## 5. 版本历史
```

**设计图放置：**
- 设计图 → `assets/`（模块根，最新全量）
- 版本快照 → `versions/vX.Y/assets/`
- Pencil 文件 → `SPEC-XXX.pen`（根，最新）

---

### 2.4 任务文档（-4）

**必含章节：**
```
# SPEC-XXX-4 任务文档
## 1. 开发目标（一句话，服务+端口）
## 2. 进度状态说明（符号表）
## 3. 任务拆解（按阶段，表格格式）
## 4. DoD（Definition of Done）清单
## 5. 版本历史
```

**任务表格列：** `任务编号 | 任务名称 | 状态 | 负责人 | 估时 | 实际用时 | 依赖 | 输出 | PR/备注`

**任务编号格式：** `TASK-{SPEC编号}-{类型}{序号}`（如 `TASK-101-D01`、`TASK-101-01`）

**状态符号：**
| 符号 | 含义 |
|------|------|
| ⬜ | 待开始 |
| 🔵 | 进行中 |
| ✅ | 已完成 |
| 🔴 | 阻塞 |

---

### 2.5 验收文档（-5）

**必含章节：**
```
# SPEC-XXX-5 验收文档（全量测试规范）
## 1. 验收范围（测试层次表）
## 2. 测试环境与账号
## 3. 接口测试规范（按 API 端点，含请求/响应/边界/错误）
## 4. 功能测试规范（端到端业务场景）
## 5. 安全性测试
## 6. UI 自动化测试（Playwright）
## 7. 性能测试指标
```

---

### 2.6 数据库文档（-6）

**必含章节：**
```
# SPEC-XXX-6 数据库文档
## 1. 涉及数据表总览（表名、说明、与本模块关系）
## 2. 主表：{表名}（完整 DDL + 字段说明 + 索引）
## 3. 关联表（关键字段说明）
## 4. 数据变更记录（ALTER TABLE 历史）
## 5. 版本历史
```

**DDL 规范：**
- 表名：`ai_*` 前缀
- 主键：`bigint NOT NULL`（雪花算法，`@TableId(ASSIGN_ID)`，**非 AUTO_INCREMENT**）
- 必须包含 BladeX BaseEntity 9 个标准字段
- 字符集：`ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci`

### 2.7 快速排查手册（-7）

**必含章节：**
```
# SPEC-XXX-7 快速排查手册
## 1. 快速诊断清单（启动检查项）

```

### 2.8 可追溯性矩阵（-8）

**必含章节：**
```
# SPEC-XXX-8 可追溯性矩阵
## 1. 矩阵说明（编号体系、状态说明）
## 2. 模块代码与符号（两位字母缩写）
## 3. 需求→API 映射表
## 4. API→代码文件 映射表
## 5. API→测试用例 映射表
## 6. 数据库表→代码 映射表
## 7. 版本历史
```

### 2.9 验收报告（-9）

**必含章节：**
```
# SPEC-XXX-9 验收报告
## 1. 验收概述（范围、方法、总体结论）
## 2. 源码文件清单与验收结论
## 3. API 验收结论（逐一列出）
## 4. DoD 核查清单
## 5. 遗留问题
## 6. 版本历史
```

## 三、通用规范

### 3.1 文件头格式
每个文档文件头必须包含：
```markdown
# SPEC-XXX-{序号} {文档类型}
> 模块：{模块名} | 服务：{服务名}:{端口} | 版本：v{X.Y} | 日期：YYYY-MM-DD
```

### 3.2 版本历史格式（每个文档必须有）
```markdown
## 版本历史
| 版本 | 日期 | 作者 | 变更说明 |
|------|------|------|----------|
| v1.0 | YYYY-MM-DD | {作者} | 初始版本 |
```

### 3.3 禁止项
- 禁止 SELECT *（SQL 示例中）
- 禁止 PostgreSQL 语法（BIGSERIAL, TIMESTAMPTZ, JSONB）
- 禁止表名用 `t_*` 前缀
- 禁止硬编码文案（文档示例中注明需走 i18n）
