## 二、中间件启动（Docker 快速方案）

```bash
# MySQL 8.0
docker run -d --name gh-mysql \
  -e MYSQL_ROOT_PASSWORD=请看凭据账号文档 \
  -e MYSQL_DATABASE=globehive \
  -p 3306:3306 \
  mysql:8.0 --character-set-server=utf8mb4

# Redis 7
docker run -d --name gh-redis \
  -p 6379:6379 \
  redis:7 --requirepass 请看凭据账号文档

# Nacos（注册中心 + 配置中心）
docker run -d --name gh-nacos \
  -e MODE=standalone \
  -p 8848:8848 \
  nacos/nacos-server:v2.3.0
```

> 账号密码见 `19-凭据账号.md`。

## 三、数据库初始化

### 3.1 创建数据库

```sql
CREATE DATABASE IF NOT EXISTS `globehive`
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_general_ci;
```

### 3.2 执行 BladeX 基础表

执行 BladeX 脚手架自带的 `blade.sql`（位于项目根目录 `doc/sql/`）。

### 3.3 执行各模块 DDL

按批次顺序执行各模块的 DDL 文件：

```bash
# 以 SPEC-101 为例
mysql -u root -p globehive < 02-modules/SPEC-101-用户画像模块/sql/SPEC-101-ddl.sql
```

> 执行顺序与模块依赖关系一致，参见 `01-roadmap/路线图.md` 的依赖图。

## 四、后端启动顺序

1. **启动中间件**：MySQL → Redis → Nacos
2. **导入 Nacos 配置**：在 Nacos 控制台导入 `doc/nacos/` 目录下的配置文件
3. **启动网关**：`globehive-gateway`
4. **启动基础服务**：`globehive-auth` → `globehive-system`
5. **启动业务服务**：按需启动对应模块的微服务

> 启动地址与端口见 `19-凭据账号.md` → 本地环境一节。

---

## 五、前端启动

```bash
cd globehive-ui
npm install
npm run dev
```

> 默认访问：`http://localhost:8080`
> API 代理目标：`http://localhost:9090`（网关地址）

## 六、本地 hosts 配置

如需通过域名访问（如 Nacos 配置了域名），在本地 hosts 文件追加：

```
127.0.0.1  nacos.globehive.local
127.0.0.1  gateway.globehive.local
```

## 八、常见问题

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 服务启动报 `NoSuchBeanDefinitionException` | Nacos 配置未导入 | 重新导入 Nacos 配置文件 |
| 登录后 401 | JWT 密钥与配置不一致 | 检查 `blade.token.sign-key` |
| 前端页面空白 | 代理配置错误 | 检查 `vite.config.js` 的 proxy 地址 |
| MySQL 连接拒绝 | 未启动或端口冲突 | `docker ps` 确认容器运行状态 |
| 踩到其他坑 | — | 记录到 `20-踩坑记录.md` |
