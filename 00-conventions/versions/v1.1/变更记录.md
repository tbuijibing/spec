# 规范体系 v1.1 变更说明

> 版本：v1.1 | 发布日期：2026-03-30 | 状态：✅ 已发布

---

## 版本目标

在 v1.0 的基础上，把 `00-conventions` 从基础规范集合升级为覆盖设计、架构、安全、测试、性能、运维、国际化、多客户端、MCP 的完整核心规范体系。

---

## 新增规范

| 规范文档 | 内容摘要 |
|----------|----------|
| `06-设计系统规范.md` | 设计系统规范：视觉语言、组件一致性、可访问性、页面结构 |
| `07-设计资产规范.md` | 设计资产规范：Figma / Pencil / assets / 版本快照 / Frame 命名 |
| `08-技术架构规范.md` | 技术架构规范：分层、微服务边界、多租户、Agent / MCP 边界 |
| `09-安全规范.md` | 安全规范：JWT、RBAC、租户隔离、数据安全、AI 安全 |
| `10-测试规范.md` | 测试规范：测试金字塔、模块必测项、验收证据、覆盖率 |
| `11-性能规范.md` | 性能规范：P95/P99 指标、阶段目标、容量与扩展要求 |
| `12-日志与监控规范.md` | 日志与监控规范：结构化日志、脱敏、指标、告警 |
| `13-错误处理规范.md` | 错误处理规范：异常分类、错误码、前后端边界 |
| `14-部署与运维规范.md` | 部署与运维规范：环境、发布前检查、回滚、备份恢复 |
| `15-国际化规范.md` | 国际化规范：三层翻译架构、语言代码、翻译表、RTL |
| `16-多客户端规范.md` | 多客户端规范：单客户端 / 双客户端 / 多客户端 specs 组织方式 |
| `17-MCP集成规范.md` | MCP 集成规范：Host / Client / Server / Tool / Resource / 协议 |
| `18-可追溯性矩阵规范.md` | 可追溯性矩阵规范：需求、设计、路由、API、代码、测试之间的映射规则 |

---

## 本次补强的关键点

| 主题 | 说明 |
|------|------|
| 设计规范正式入库 | 设计系统与设计资产被纳入 `00-conventions`，不再只是外围前端文档 |
| 核心开发规范闭环 | 架构、安全、测试、性能、日志、错误处理、运维形成完整开发约束 |
| 国际化与多端前置 | i18n 与多客户端结构不再等项目后期再补 |
| MCP 规范落地 | AI 外部工具集成从“想法”上升为正式规范 |
| 可追溯性前置 | 需求、设计、接口、代码、测试的映射规则被正式纳入项目级规范 |
