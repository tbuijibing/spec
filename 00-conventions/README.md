## 规范复查机制

| 项目 | 规则 |
|------|------|
| 复查周期 | 每个版本迭代结束后触发一次规范复查 |
| 复查范围 | 本次迭代中被频繁引用或发生踩坑的规范文档 |
| 负责人 | 项目负责人（或发现问题的开发人员提 PR） |
| 复查结果 | 更新对应规范文档版本号 + 在本表规范迭代历史中记录 |
| 过时规范处理 | 在文档顶部加 `> ⚠️ 该规范已过时，以 vX.Y 为准` 提示，保留文件不删除 |
