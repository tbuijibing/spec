# 规范体系 v1.0 变更说明

> 版本：v1.0 | 发布日期：2026-03-30 | 状态：✅ 已发布

---

## 版本目标

建立 GlobeHive AI 项目完整的开发规范体系，覆盖文档编写、版本管理、命名、数据库、API 五个方向。

---

## 新增规范

| 规范文档 | 内容摘要 |
|----------|----------|
| `01-SPEC文档编写规范.md` | 9 类 SPEC 文档格式（需求/技术/UI/任务/验收/DB/排查/矩阵/报告），含用户故事模板、任务表格格式 |
| `02-版本管理规范.md` | 三层版本号（vX.Y / vX.Y.Z / vX.Y.Z.NN），完整迭代工作流，`versions/` 目录结构 |
| `03-命名规范.md` | Java 包名/类名/方法名命名，MySQL 表名/字段名/索引命名，Vue 组件/API 函数命名，文件目录命名 |
| `04-数据库开发规范.md` | MySQL 8.0 规范，BladeX BaseEntity 9 个标准字段，数据类型选择表，DDL 模板，翻译表规范 |
| `05-接口设计规范.md` | RESTful URL 设计，R<T> 统一返回，分页查询规范，Controller 代码模板，前端 API 模块规范 |

---

## 核心决策

| 决策 | 说明 |
|------|------|
| 禁止 PostgreSQL 语法 | 项目数据库为 MySQL 8.0 |
| 禁止 `t_` 表前缀 | 统一用 `ai_` 前缀 |
| 禁止 AUTO_INCREMENT 主键 | 统一雪花算法 `@TableId(ASSIGN_ID)` |
| 禁止 SELECT * | 所有 SQL 明确列名 |
| 统一 `R<T>` 返回 | 所有 Controller 返回值统一包装 |
