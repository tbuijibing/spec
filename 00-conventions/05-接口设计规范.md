# API 设计规范

> 适用范围：所有 GlobeHive AI 后端 Controller 及前端 API 调用

---

## 一、URL 设计

### 1.1 基本原则

| 规则 | 正确 | 错误 |
|------|------|------|
| 资源名用复数名词 | `/user-profiles` | `/getUserProfile` |
| 连字符分隔多词 | `/user-profiles` | `/userProfiles` |
| 不在 URL 中放动词 | `DELETE /user-profiles/{id}` | `/deleteUserProfile` |
| 嵌套资源表示从属 | `/user-profiles/{id}/tags` | `/getUserTags?userId=` |

### 1.2 URL 模板

| 操作 | HTTP 方法 | URL 模式 | 示例 |
|------|-----------|---------|------|
| 分页查询 | `GET` | `/{resource}/page` | `GET /user-profiles/page` |
| 查询详情 | `GET` | `/{resource}/{id}` | `GET /user-profiles/1` |
| 新增 | `POST` | `/{resource}` | `POST /user-profiles` |
| 更新 | `PUT` | `/{resource}/{id}` | `PUT /user-profiles/1` |
| 删除 | `DELETE` | `/{resource}/{id}` | `DELETE /user-profiles/1` |
| 批量删除 | `POST` | `/{resource}/remove` | `POST /user-profiles/remove` |
| 批量操作 | `POST` | `/{resource}/batch-{action}` | `POST /user-profiles/batch-import` |
| 子资源操作 | `POST`/`DELETE` | `/{resource}/{id}/{sub}` | `POST /user-profiles/1/tags` |

## 二、统一返回结构 `R<T>`

所有接口统一使用 BladeX `R<T>` 包装：

```json
{
  "code": 200,
  "success": true,
  "msg": "操作成功",
  "data": { ... }
}
```

| 场景 | Java 写法 | code |
|------|-----------|------|
| 成功有数据 | `R.data(result)` | 200 |
| 成功无数据 | `R.status(true)` | 200 |
| 业务失败 | `R.fail("错误信息")` | 400 |
| 未授权 | 框架自动处理 | 401 |
| 禁止访问 | 框架自动处理 | 403 |
| 服务异常 | 框架全局异常处理 | 500 |

## 四、Controller 规范

```java
@RestController
@RequestMapping("/user-profiles")
@AllArgsConstructor
public class AiUserProfileController {

    private final IAiUserProfileService userProfileService;

    @GetMapping("/page")
    @ApiOperation("用户画像分页查询")
    public R<IPage<AiUserProfileVO>> page(UserProfileQuery query) {
        return R.data(userProfileService.pageProfile(query));
    }

    @GetMapping("/{id}")
    @ApiOperation("用户画像详情")
    public R<AiUserProfileVO> detail(@PathVariable Long id) {
        return R.data(userProfileService.getProfileById(id));
    }

    @PostMapping
    @ApiOperation("新增用户画像")
    public R<Boolean> save(@RequestBody @Valid AiUserProfileDTO dto) {
        return R.status(userProfileService.saveProfile(dto));
    }

    @PutMapping("/{id}")
    @ApiOperation("更新用户画像")
    public R<Boolean> update(@PathVariable Long id, @RequestBody @Valid AiUserProfileDTO dto) {
        dto.setId(id);
        return R.status(userProfileService.updateProfile(dto));
    }

    @DeleteMapping("/{id}")
    @ApiOperation("删除用户画像")
    public R<Boolean> remove(@PathVariable Long id) {
        return R.status(userProfileService.removeById(id));
    }
}
```

## 五、错误码规范

| 范围 | 说明 | 示例 |
|------|------|------|
| `200` | 成功 | — |
| `400xx` | 业务错误（参数/逻辑）| `40001`=参数缺失 |
| `401xx` | 未授权 | 框架处理 |
| `403xx` | 权限不足 | 框架处理 |
| `404xx` | 资源不存在 | `40401`=用户不存在 |
| `500xx` | 服务内部错误 | 框架全局异常处理 |

## 六、前端 API 模块规范

```javascript
// src/api/globehive/base/index.js
import request from '@/router/axios';

// 分页查询
export const pageUserProfile = (params) =>
  request({ url: '/globehive-base/user-profiles/page', method: 'get', params });

// 查询详情
export const getUserProfile = (id) =>
  request({ url: `/globehive-base/user-profiles/${id}`, method: 'get' });

// 新增
export const saveUserProfile = (data) =>
  request({ url: '/globehive-base/user-profiles', method: 'post', data });

// 更新
export const updateUserProfile = (id, data) =>
  request({ url: `/globehive-base/user-profiles/${id}`, method: 'put', data });

// 删除
export const removeUserProfile = (id) =>
  request({ url: `/globehive-base/user-profiles/${id}`, method: 'delete' });
```
