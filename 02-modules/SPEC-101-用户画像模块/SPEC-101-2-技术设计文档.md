## 1. 架构设计

### 1.1 分层架构

```text
Controller Layer
  └─ AiUserProfileController
       ├─ page()        GET  /user-profiles/page
       ├─ detail()      GET  /user-profiles/{id}
       ├─ save()        POST /user-profiles
       ├─ update()      PUT  /user-profiles/{id}
       ├─ remove()      DEL  /user-profiles/{id}
       ├─ addTags()     POST /user-profiles/{id}/tags
       ├─ removeTags()  DEL  /user-profiles/{id}/tags
       ├─ merge()       POST /user-profiles/merge
       ├─ batchImport() POST /user-profiles/batch-import
       ├─ template()    GET  /user-profiles/template
       └─ export()      GET  /user-profiles/export
            ↓
Service Layer
  ├─ IAiUserProfileService
  └─ AiUserProfileServiceImpl
       ├─ 画像聚合查询（JOIN chat_user_relation）
       ├─ 增量更新（selectiveUpdate）
       ├─ 标签去重合并
       ├─ 完整度计算（0-100 分）
       ├─ ONE ID 合并（事务）
       └─ 审计日志写入（@Async）
            ↓
Mapper Layer
  └─ AiUserProfileMapper（XML + MyBatis-Plus）
            ↓
Entity Layer
  └─ AiUserProfile（ai_user_profile，JSON 字段用 JacksonTypeHandler）
```

### 1.2 分层职责

| 层 | 继承 | 职责 |
|----|------|------|
| Controller | — | 参数校验、权限检查、R<T> 返回 |
| Service 接口 | `IService<AiUserProfile>` | 接口定义 |
| Service 实现 | `ServiceImpl<M,T>` | 业务逻辑、事务边界 |
| Mapper | `BaseMapper<AiUserProfile>` | SQL 映射、XML 查询 |

## 2. 关键数据流

### 2.1 画像查询流程

```text
GET /user-profiles/page
  → Controller.page(query)
  → Service.pageProfile(query)          // 分页
  → Mapper.pageProfile(query, page)     // LEFT JOIN ai_chat_user_relation
  → MyBatis XML resultMap               // JSON 字段用 JacksonTypeHandler
  → List<AiUserProfileVO>               // VO 包含渠道信息
  → R.data(page)
```

### 2.2 ONE ID 合并流程

```text
POST /user-profiles/merge
  → Controller.merge(mainId, mergedId)
  → @Transactional Service.mergeProfiles()
      1. 校验：主账号不能是 status=2（已被合并）
      2. 查被合并账号的 ai_chat_user_relation 记录
      3. UPDATE ai_chat_user_relation SET user_id=mainId WHERE user_id=mergedId
      4. UPDATE ai_user_profile SET status=2 WHERE id=mergedId
      5. @Async 写审计日志
  → R.status(true)
```

### 2.3 完整度计算算法

```text
基础字段（×1.0 权重）：real_name, gender, birthday, constellation, profession, income_range
扩展字段（×0.5 权重）：family_members, anniversaries, addresses, social_accounts, preferences

已填字段数 × 对应权重 / 总权重 × 100 = 完整度（向下取整）
上限：100
```

## 4. 关键实现细节

### 4.1 JSON 字段处理（避免 500 错误）

```java
// Entity 字段上加 JacksonTypeHandler
@TableField(value = "family_members", typeHandler = JacksonTypeHandler.class)
private Object familyMembers;

// Mapper XML resultMap 显式声明 typeHandler
<result column="family_members" property="familyMembers"
        typeHandler="com.baomidou.mybatisplus.extension.handlers.JacksonTypeHandler"/>
```

### 4.2 完整度字段自动更新

```java
// Service 中每次 update 后调用
private Integer calcCompleteness(AiUserProfile profile) {
    int filled = 0, total = 20;
    if (StringUtils.isNotBlank(profile.getRealName())) filled += 2;
    // ... 每个字段检查
    return (int)(filled * 100.0 / total);
}
```
