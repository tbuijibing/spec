# 命名规范

> 适用范围：GlobeHive AI 全栈（Java 后端 / MySQL / Vue 前端 / 文件目录）

---

## 一、Java 后端命名

### 1.1 包名

| 层级 | 规则 | 示例 |
|------|------|------|
| 根包 | `com.globehive.{module}` | `com.globehive.base` |
| Controller | `com.globehive.{module}.controller` | `com.globehive.base.controller` |
| Service 接口 | `com.globehive.{module}.service` | `com.globehive.base.service` |
| Service 实现 | `com.globehive.{module}.service.impl` | `com.globehive.base.service.impl` |
| Mapper | `com.globehive.{module}.mapper` | `com.globehive.base.mapper` |
| Entity | `com.globehive.{module}.pojo.entity` | `com.globehive.base.pojo.entity` |
| VO/DTO | `com.globehive.{module}.pojo.vo` | `com.globehive.base.pojo.vo` |

### 1.2 类名

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 实体类 | `Ai` + 名词（PascalCase） | `AiUserProfile` |
| Service 接口 | `I` + 名词 + `Service` | `IAiUserProfileService` |
| Service 实现 | 名词 + `ServiceImpl` | `AiUserProfileServiceImpl` |
| Controller | 名词 + `Controller` | `AiUserProfileController` |
| Mapper | 名词 + `Mapper` | `AiUserProfileMapper` |
| VO | 名词 + `VO` | `AiUserProfileVO` |
| Query | 名词 + `Query` | `UserProfileQuery` |

### 1.3 方法名

| 操作 | 命名前缀 | 示例 |
|------|----------|------|
| 查询单条 | `get` / `find` | `getUserProfile()` |
| 查询列表 | `list` / `query` | `listUserProfiles()` |
| 分页查询 | `page` | `pageUserProfile()` |
| 新增 | `save` / `create` | `saveUserProfile()` |
| 更新 | `update` | `updateUserProfile()` |
| 删除 | `remove` / `delete` | `removeUserProfile()` |
| 批量 | `batch` 前缀 | `batchImportProfiles()` |

### 1.4 返回值

所有 Controller 方法统一返回 `R<T>`：
```java
R.data(result)     // 成功有数据
R.status(true)     // 成功无数据
R.fail("msg")      // 业务失败
```

## 二、数据库命名

### 2.1 表名

| 来源 | 前缀 | 示例 |
|------|------|------|
| BladeX 框架表 | `blade_` | `blade_user`, `blade_tenant` |
| GlobeHive 业务表 | `ai_` | `ai_user_profile`, `ai_session` |
| 多语言翻译表 | `{原表名}_translate` | `ai_user_profile_translate` |

> **禁止** 使用 `t_` 前缀（历史遗留错误，已废弃）

### 2.2 字段名

| 规则 | 说明 | 正确示例 | 错误示例 |
|------|------|----------|----------|
| 蛇形命名 | 全小写下划线 | `user_id` | `userId` |
| 主键 | 统一 `id` | `id bigint NOT NULL` | `user_id` 作主键 |
| 外键引用 | `{被引用表}_id` | `user_id`, `tenant_id` | `uid` |
| 时间字段 | `create_time`, `update_time` | — | `created_at`, `add_time` |
| 状态字段 | `status`（int）| `status int DEFAULT 1` | `is_active` |
| 软删除 | `is_deleted`（int）| `is_deleted int DEFAULT 0` | `deleted` |

### 2.3 索引名

| 类型 | 前缀 | 示例 |
|------|------|------|
| 唯一索引 | `uk_` | `uk_tenant_user` |
| 普通索引 | `idx_` | `idx_tenant_id` |
| 全文索引 | `ft_` | `ft_message_content` |

## 三、Vue 前端命名

### 3.1 目录与文件

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 页面组件 | camelCase + `.vue` | `userProfile.vue` |
| API 模块 | `index.js` | `src/api/globehive/base/index.js` |
| 业务页面目录 | `src/views/globehive/{service}/` | `src/views/globehive/base/` |

### 3.2 API 函数名

| 操作 | 命名 | 示例 |
|------|------|------|
| 分页查询 | `page{Entity}` | `pageUserProfile(params)` |
| 查询详情 | `get{Entity}` | `getUserProfile(id)` |
| 新增 | `save{Entity}` | `saveUserProfile(data)` |
| 更新 | `update{Entity}` | `updateUserProfile(data)` |
| 删除 | `remove{Entity}` | `removeUserProfile(id)` |

## 四、文件目录命名

| 目录/文件 | 命名规则 | 示例 |
|-----------|----------|------|
| SPEC 模块目录 | `SPEC-{N}-{中文模块名}` | `SPEC-101-用户画像模块` |
| 设计图目录 | 固定 `assets/` | `assets/` |
| 测试产物目录 | 固定 `test-report/` | `test-report/` |
| 版本归档目录 | `versions/vX.Y.Z/` | `versions/v1.0/` |
| Pencil 文件（根） | `SPEC-{N}.pen` | `SPEC-101.pen` |
| Pencil 文件（版本）| `SPEC-{N}-v{X.Y.Z}.pen` | `SPEC-101-v1.0.pen` |
| SQL 文件 | `{模块}.{db}.{操作}.sql` | `base.mysql.create.sql` |
| 截图 | `{02d}-{英文描述}.png` | `01-user-list.png` |
