# AI 质量规范

> 适用范围：所有 {项目名} 功能模块 | 更新：2026-03-30

---

## 一、规范目标

| 目标 | 说明 |
|------|------|
| 输出可评估 | AI 响应有明确的 Pass/Fail 判定标准 |
| Prompt 可追溯 | Prompt 变更有版本记录，可回滚 |
| 成本可控 | Token 用量有监控、有告警阈值 |
| 降级有预案 | 主模型不可用时有明确的 fallback 策略 |

## 二、Prompt 版本管理

### 2.1 存放位置

每个模块的 Prompt 文件存放在模块目录下：

```
SPEC-XXX-模块名/
└── prompts/
    ├── system-prompt-v1.0.txt   ← 系统 Prompt
    ├── user-template-v1.0.txt   ← 用户消息模板
    └── 变更记录.md               ← Prompt 版本日志
```

### 2.2 命名规则

```
{用途}-v{X.Y}.txt
```

| 示例 | 说明 |
|------|------|
| `system-prompt-v1.0.txt` | 系统级 Prompt，v1.0 |
| `classify-template-v2.1.txt` | 分类任务模板，v2.1 |

### 2.3 变更记录格式

```markdown
## v1.1 — 2026-04-01

**变更原因**：原版本对多语言用户识别率低
**变更内容**：在 system prompt 中增加语言识别指令
**测试结果**：多语言准确率 +12%
**发布者**：张三
```

### 2.4 禁止项

- 禁止直接修改线上 Prompt 而不更新版本号。
- 禁止把用户输入直接拼接进高权限系统 Prompt（Prompt Injection 风险）。
- 禁止删除旧版本 Prompt 文件（保留历史，支持回滚）。

## 三、AI 输出质量评估标准

### 3.1 通用评估维度

| 维度 | 说明 | 评估方式 |
|------|------|----------|
| 相关性 | 回答是否切题 | 人工抽样 / 自动评分 |
| 准确性 | 信息是否正确 | 对照知识库校验 |
| 安全性 | 是否包含违禁内容 | 规则过滤 + 人工审查 |
| 格式合规 | 是否符合预期输出格式 | 自动解析验证 |
| 延迟 | 首 Token 响应时间 | P95 ≤ 2s |

### 3.2 模块验收时的 AI 质量门禁

每个含 AI 功能的模块，验收前必须通过：

| 检查项 | 标准 |
|--------|------|
| 正常场景准确率 | ≥ 85% |
| 边界输入（空/超长/乱码）不崩溃 | 100% |
| 敏感词过滤生效 | 测试用例全部拦截 |
| 响应延迟 P95 | ≤ 2 秒（流式输出除外） |
| 越权 Prompt 注入测试 | 全部拒绝 |

### 3.3 评估记录存放

```
SPEC-XXX-5-验收文档.md  ← AI 质量测试用例
SPEC-XXX-9-验收报告.md  ← AI 质量验收结论
```

## 四、Token 用量管理

### 4.1 单次请求限制

| 场景 | 上限 |
|------|------|
| 普通对话 | 4,096 tokens（输入 + 输出合计） |
| 知识库召回增强（RAG） | 8,192 tokens |
| 长文本分析 | 32,768 tokens（需走异步接口） |

### 4.2 监控告警阈值

| 指标 | 告警阈值 |
|------|----------|
| 单租户日 Token 用量 | > 100 万 tokens |
| 单接口平均 Token | > 2,000 tokens |
| 总费用日环比 | 增长 > 50% |

> 监控看板配置见运维规范（`14-部署与运维规范.md`）。

## 五、大模型降级策略

### 5.1 降级触发条件

- 主模型接口响应时间 > 10 秒
- 主模型接口连续失败 3 次
- 主模型服务商宣告故障

### 5.2 降级方案

```
主模型（GPT-4o / Claude 3.5）
  ↓ 降级
备用模型（GLM-4 / Qwen-Max）
  ↓ 降级
本地模型 / 规则兜底（仅返回固定提示，不做 AI 推理）
```

### 5.3 降级行为规范

- **降级时必须记录日志**：标注触发原因、降级层级、用户 ID。
- **前端无感知**：降级不能导致前端报错，最多降低回复质量。
- **恢复时自动切回**：主模型恢复后，新请求自动切回，无需重启。
