## 3. 接口测试规范

### 3.1 分页查询 GET /user-profiles/page

| 测试用例 | 请求 | 预期结果 |
|----------|------|----------|
| 正常分页 | `?current=1&size=10` | 200，records 数组，total >= 0 |
| 关键词搜索 | `?keyword=张三` | 200，records 中 real_name 含"张三" |
| 越界页码 | `?current=9999` | 200，records 为空数组 |
| 未授权 | 无 token | 401 |
| 跨租户 | 用租户A账号查全局 | 只返回租户A数据 |

### 3.2 创建画像 POST /user-profiles

| 测试用例 | 请求体 | 预期结果 |
|----------|--------|----------|
| 正常创建 | `{"userId":1,"realName":"张三"}` | 200，data=true |
| 缺少必填字段 | `{}` | 400，提示缺少 userId |
| 完整度计算 | 填写 10 个字段 | 返回后查詢 completeness > 0 |

### 3.3 ONE ID 合并 POST /user-profiles/merge

| 测试用例 | 请求体 | 预期结果 |
|----------|--------|----------|
| 正常合并 | `{"mainId":1,"mergedId":2}` | 200，data=true |
| 主账号是已合并账号 | mainId 的 status=2 | 400，提示不能合并 |
| 两个 ID 相同 | mainId == mergedId | 400 |

## 4. 功能测试规范

### 4.1 完整 CRUD 流程

```
1. 打开 http://localhost:8000/globehive/base/userProfile
2. 点击"新增" → 填写基础信息（姓名/性别/生日）→ 保存
3. 列表中可见新记录
4. 点击"编辑" → 修改姓名 → 保存
5. 列表中显示修改后的姓名
6. 点击"删除" → 确认
7. 记录消失（软删除，DB 中 is_deleted=1）
```

### 4.2 标签管理流程

```
1. 点击某用户的"标签"按钮
2. 添加标签"VIP"→ 保存
3. 再次打开：显示"VIP"标签
4. 添加已有标签"VIP"→ 保存（不重复）
5. 删除"VIP" → 保存 → 标签消失
```

### 4.3 批量导入流程

```
1. 下载 Excel 模板
2. 填写 10 条用户数据
3. 上传模板文件
4. 查看导入结果（10 条成功）
5. 在列表中搜索到导入的用户
```

## 5. 安全性测试

| 测试项 | 操作 | 预期 |
|--------|------|------|
| 租户隔离 | 用租户A账号访问租户B的 /user-profiles/{id} | 403 或空数据 |
| SQL 注入 | keyword 参数注入 `' OR 1=1--` | 正常搜索，无 SQL 错误 |
| 未授权访问 | 不带 token 访问任何接口 | 401 |
| 越权修改 | 普通客服修改他人数据 | 403 |
