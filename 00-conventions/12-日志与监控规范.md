# 日志与监控规范

> 来源：`plans/监控与运维方案.md`

---

## 一、日志级别

| 级别 | 用途 |
|------|------|
| DEBUG | 开发调试 |
| INFO | 正常业务记录 |
| WARNING | 需要关注但未失败 |
| ERROR | 业务失败或系统异常 |
| CRITICAL | 影响可用性的严重故障 |

---

## 二、日志类型

| 类型 | 说明 |
|------|------|
| 业务日志 | 关键业务动作 |
| 错误日志 | 异常、失败、报警 |
| 审计日志 | 登录、权限、导出、敏感操作 |
| 接入日志 | API 请求入口、耗时、调用方 |

---

## 三、结构化日志字段

建议至少包含：
- `timestamp`
- `level`
- `service`
- `module`
- `action`
- `tenant_id`
- `user_id`
- `trace_id`
- `message`

---

## 四、脱敏要求

- 密码、Token、密钥不得入日志。
- 手机、邮箱、身份证等敏感信息必须脱敏。
- AI 原始 Prompt / 用户敏感输入默认不可直接落日志。

---

## 五、监控指标

| 范围 | 指标 |
|------|------|
| 应用 | QPS、响应时间、错误率、CPU、内存 |
| 数据库 | 连接数、慢查询、锁等待、缓冲池命中率 |
| Redis | 命中率、延迟、内存使用 |
| MQ / 异步任务 | 堆积数、消费延迟、失败数 |

---

## 六、告警要求

- ERROR / CRITICAL 必须可检索。
- 核心服务不可用必须告警。
- 数据库慢查询和锁等待需设阈值。
- 同一错误高频重复出现要聚合告警。

---

## 七、模块文档要求

模块排查手册 `SPEC-XXX-7` 必须说明：
- 关键日志位置
- 常见错误关键字
- 基础诊断命令
- 监控指标关注点
