# MCP 集成规范

> 来源：`plans/specs/需求规范-12-MCP集成架构.md`

---

## 一、角色定义

| 角色 | 说明 |
|------|------|
| MCP Host | 承载 AI Agent 的宿主应用（GlobeHive AI） |
| MCP Client | 负责发现、连接、调用 MCP Server |
| MCP Server | 对外暴露 Tool / Resource / Prompt |

---

## 二、设计原则

- Tool 必须有清晰输入输出和权限边界。
- Resource 必须可追溯到真实数据源。
- Prompt 模板必须版本化管理。
- 外部 Server 的接入协议必须标准化。

---

## 三、传输协议

| 场景 | 协议 |
|------|------|
| 本地 / 内置服务 | `stdio` |
| 远程服务 | `SSE` |
| 企业自建 / 标准集成 | `HTTP + OAuth 2.1` |

---

## 四、Tool 设计要求

每个 Tool 至少应定义：
- Tool 名称
- 功能描述
- 参数结构
- 返回结构
- 权限要求
- 失败场景
- 审计要求

---

## 五、接入约束

| 项目 | 要求 |
|------|------|
| 权限 | Tool 调用必须带用户/租户上下文 |
| 审计 | 关键 Tool 调用要有审计日志 |
| 幂等性 | 写操作 Tool 需说明幂等策略 |
| 失败处理 | 超时、重试、降级要明确 |
| 安全 | 外部输入按不可信处理 |

---

## 六、模块文档要求

涉及 MCP 的模块，必须在技术设计文档中说明：
- MCP 角色位置
- 调用链路
- Tool / Resource 清单
- 安全边界
- 测试方式

---

## 七、禁止项

- 禁止把 MCP 当作无约束的任意调用总线。
- 禁止没有权限校验就暴露 Tool。
- 禁止外部资源接入后无可追溯记录。
