# 版本管理规范

> 适用范围：所有 `SPEC-XXX-模块名/versions/` 目录

---

## 一、版本号命名规范

| 层级 | 格式 | 含义 | 示例 |
|------|------|------|------|
| 主版本 | `vX.Y` | 大功能里程碑 / 正式发布 | `v1.0`, `v1.1`, `v2.0` |
| 增量版本 | `vX.Y.Z` | 当前主版本的小功能或 Bug 修复迭代 | `v1.0.1`, `v1.0.2` |
| 测试轮次 | `vX.Y.Z.NN` | 增量版本测试过程中的每轮（双位数字） | `v1.0.1.01`, `v1.0.1.02` |

> **规则**：测试轮次 `vX.Y.Z.NN` 仅用于 `test-report/` 子目录命名，不对外发布。

---

## 二、完整目录结构

```text
SPEC-XXX-模块名/
│
├── ── 根目录：始终保持最新全量 ──────────────────
├── SPEC-XXX-1-需求文档.md        ← 最新全量（迭代后就地更新）
├── SPEC-XXX-2~8-*.md
├── SPEC-XXX-9-验收报告.md
├── SPEC-XXX.pen                  ← 最新 Pencil 文件（全量）
├── assets/                       ← 最新全量设计图
│
└── ── versions/：每次发版后归档 ─────────────────
    versions/
    ├── v1.0/                          ← 初始正式版
    │   ├── 变更记录.md                 ← 整体变动说明（7章）
    │   ├── assets/                    ← v1.0 设计图增量
    │   ├── SPEC-XXX-v1.0.pen          ← v1.0 Pencil 快照
    │   └── test-report/
    │       └── SPEC-XXX-v1.0-测试报告.md
    │
    ├── v1.0.1/                        ← 增量版本
    │   ├── 变更记录.md
    │   ├── assets/
    │   ├── SPEC-XXX-v1.0.1.pen
    │   └── test-report/
    │       ├── SPEC-XXX-v1.0.1-测试报告.md  ← 最终通过报告
    │       ├── v1.0.1.01/             ← 第1轮（发现 bug）
    │       │   ├── SPEC-XXX-v1.0.1.01-测试记录.md
    │       │   └── *.png
    │       └── v1.0.1.02/             ← 第2轮（修复后通过）
    │           ├── SPEC-XXX-v1.0.1.02-测试记录.md
    │           └── *.png
    └── v1.1/                          ← 下次迭代时创建
```

## 三、四层内容职责

| 位置 | 内容 | 更新时机 |
|------|------|----------|
| 根目录 spec 文档 | 全量最新 | 每次发版后就地更新 |
| 根目录 `assets/` | 全量最新设计图 | 有 UI 变动时替换 |
| 根目录 `SPEC-XXX.pen` | 全量最新 Pencil 文件 | 有设计变动时覆盖 |
| `versions/vX.Y/变更记录.md` | 本版变更说明 | 版本归档时填写 |
| `versions/vX.Y/assets/` | 本版设计图增量 | 版本归档时归入 |
| `versions/vX.Y/*.pen` | 本版 Pencil 快照 | 版本归档时复制 |
| `versions/vX.Y/test-report/` | 本版最终测试报告 | 测试通过后填写 |
| `versions/vX.Y/test-report/vX.Y.Z.NN/` | 测试轮次记录 | 每轮测试后填写 |

---

## 四、迭代工作流

```text
┌─ 开始新迭代（如 v1.0.1）
├─ 1. 创建 versions/v1.0.1/
├─ 2. 写 变更记录.md（目标 + 需求变更 + 功能列表）
├─ 3. 设计变动 → 更新根 assets/ + SPEC-XXX.pen
│         → 同步归档到 versions/v1.0.1/assets/ 和 *.pen
├─ 4. 更新根目录所有 spec 文档（反映最新状态）
├─ 5. 测试第1轮 → 创建 test-report/v1.0.1.01/
│         → 写测试记录 + 截图 → 发现 bug → 修复
├─ 6. 测试第N轮直到全部通过
│         → 写最终报告 SPEC-XXX-v1.0.1-测试报告.md
└─ 7. 更新 变更记录.md 测试结论 → 归档完成
```

## 五、文件命名规则

| 文件类型 | 命名格式 | 示例 |
|----------|----------|------|
| 变更说明 | `变更记录.md`（固定名称） | `变更记录.md` |
| Pencil 文件（根） | `SPEC-{N}.pen` | `SPEC-101.pen` |
| Pencil 文件（版本） | `SPEC-{N}-v{X.Y.Z}.pen` | `SPEC-101-v1.0.pen` |
| 正式测试报告 | `SPEC-{N}-v{X.Y.Z}-测试报告.md` | `SPEC-101-v1.0-测试报告.md` |
| 测试轮次记录 | `SPEC-{N}-v{X.Y.Z.NN}-测试记录.md` | `SPEC-101-v1.0.1.01-测试记录.md` |
| 截图 | `{02d}-{英文描述}.png` | `01-user-list.png` |
