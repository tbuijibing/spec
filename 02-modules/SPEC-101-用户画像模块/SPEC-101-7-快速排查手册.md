## 2. 常见错误排查

### ERR-101-01 分页接口返回 500（JSON 字段映射失败）

**现象**：`GET /user-profiles/page` 返回 500，日志报 `Could not read JSON`

**原因**：MyBatis resultMap 未显式声明 JSON 字段的 `JacksonTypeHandler`

**排查步骤**：
1. 检查 `AiUserProfileMapper.xml` 的 resultMap 是否包含所有 JSON 字段（`tags`, `family_members` 等）
2. 确认每个 JSON 字段的 `<result>` 标签有 `typeHandler="com.baomidou.mybatisplus.extension.handlers.JacksonTypeHandler"`

**解决方案**：
```xml
<result column="tags" property="tags"
        typeHandler="com.baomidou.mybatisplus.extension.handlers.JacksonTypeHandler"/>
<result column="family_members" property="familyMembers"
        typeHandler="com.baomidou.mybatisplus.extension.handlers.JacksonTypeHandler"/>
```

---

### ERR-101-02 前端页面 403 或数据为空

**现象**：前端列表加载后显示空数据或报 403

**原因**：用户 token 的 `tenant_id` 与查询数据不匹配，或菜单权限未配置

**排查步骤**：
1. F12 → Network → 查看接口请求头中的 `Blade-Auth` token
2. 解码 token 确认 `tenant_id` 是否正确
3. 检查 BladeX 后台菜单权限：系统管理 → 菜单管理 → `/globehive/base/userProfile` 是否存在且已分配给当前角色

**解决方案**：
- 若 token 无误，检查菜单接口 `/blade-system/menu/routes` 是否返回该路由
- 若数据为空，确认 MySQL 中 `ai_user_profile.tenant_id` 与当前用户一致

---

### ERR-101-03 ONE ID 合并失败（事务回滚）

**现象**：`POST /user-profiles/merge` 返回 500，日志报事务回滚

**原因**：`ai_chat_user_relation` 表不存在，或 UPDATE 语句字段错误

**排查步骤**：
```sql
-- 确认关联表存在
SHOW TABLES LIKE 'ai_chat_user_relation';

-- 确认关联表有 user_id 字段
DESC ai_chat_user_relation;
```

**解决方案**：先执行 SPEC-102 中的 DDL（`ai_chat_user_relation` 表建立），再重试

---

### ERR-101-04 批量导入失败（Excel 解析错误）

**现象**：上传 Excel 后返回全部失败，错误信息为"列不匹配"

**原因**：Excel 模板列顺序与代码中 `@ExcelProperty` 注解不一致

**排查步骤**：
1. 下载最新模板（`GET /user-profiles/template`），对比上传文件的列顺序
2. 检查 `AiUserProfileExcelVO.java` 中 `@ExcelProperty` 的 `index` 或 `value`

---

### ERR-101-05 完整度始终为 0

**现象**：更新画像后完整度仍为 0

**原因**：`updateProfile` 方法中未调用 `calcCompleteness()`，或 UPDATE 语句未包含 `completeness` 字段

**排查步骤**：
1. 在 Service Impl 的 `updateProfile()` 中加断点，确认 `completeness` 字段被设置
2. 检查 MyBatis Plus 的 `updateById()` 是否忽略了 null 字段（需显式设置）

## 3. 常用诊断命令

```sql

SELECT VERSION();

-- 查看最近的画像变更（排查数据异常）
SELECT id, user_id, real_name, completeness, update_time
FROM ai_user_profile ORDER BY update_time DESC LIMIT 10;

-- 查看审计日志（ONE ID 合并记录）
SELECT * FROM ai_audit_log
WHERE object_type='user_profile' ORDER BY create_time DESC LIMIT 20;
```
