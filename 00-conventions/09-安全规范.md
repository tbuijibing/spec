# 安全规范

> 来源：`plans/安全性设计文档.md`

---

## 一、安全目标

| 目标 | 说明 |
|------|------|
| 身份可信 | 接口必须具备可靠鉴权机制 |
| 权限可控 | 角色与租户边界必须清晰 |
| 数据安全 | 敏感数据可控、可审计、可脱敏 |
| AI 安全 | 提示词注入、越权调用、敏感输出必须被控制 |

---

## 二、认证与授权

### 2.1 认证

| 场景 | 机制 |
|------|------|
| Web / API 登录 | JWT + Refresh Token |
| 第三方接入 | OAuth 2.0 / OAuth 2.1 |
| 服务间调用 | 受控 API Key / 服务鉴权 |

### 2.2 授权

- 统一采用 **RBAC**。
- 角色权限必须与菜单、按钮、接口权限一致。
- `tenant_id` 是强制隔离边界，不允许在业务层忽略。

---

## 三、输入与接口安全

| 风险 | 防护要求 |
|------|----------|
| SQL 注入 | 参数化查询，禁止字符串拼 SQL |
| XSS | 输出编码，富文本白名单过滤 |
| CSRF | Web 端敏感操作需有防护机制 |
| 越权访问 | 每个资源访问都做角色/租户校验 |
| 文件上传 | 校验类型、大小、后缀、病毒风险 |

---

## 四、数据安全

| 项目 | 规范 |
|------|------|
| 密码 | 单向哈希，禁止可逆存储 |
| 敏感字段 | 按需加密存储 |
| 日志 | 敏感字段脱敏，不写明文凭证 |
| 导出 | 权限校验 + 操作审计 |
| 备份 | 有保留周期、恢复流程、访问控制 |

---

## 五、AI 安全

| 风险 | 规范 |
|------|------|
| Prompt Injection | 外部输入不得直接拼接进高权限系统 Prompt |
| 敏感输出 | 必须经过输出审查或规则控制 |
| 工具越权 | Tool 调用必须带权限和上下文校验 |
| 超范围回答 | 必须能识别并拒绝超出服务边界的问题 |

---

## 六、安全审计

以下操作必须写审计日志：
- 登录 / 登出
- 权限变更
- 数据导出
- 敏感数据修改
- Agent 上线 / 下线
- MCP 外部工具调用关键行为

---

## 七、模块文档要求

模块 `SPEC-XXX-5-验收文档.md` 中必须至少包含：
- 未授权访问测试
- 跨租户访问测试
- SQL 注入测试
- 权限边界测试
- 敏感操作审计验证

---

## 八、禁止项

- 禁止硬编码密钥、Token、账号密码。
- 禁止用前端可见字段判断权限。
- 禁止把租户隔离作为“约定”，而不是显式逻辑。
- 禁止把 AI 输出默认视为可信。

---

## 九、安全验收门禁

每个模块完成功能验收后，必须额外通过以下安全检查，否则不得标记为 `✅ 已验收`：

| 检查项 | 执行方式 | 通过标准 |
|--------|----------|----------|
| 未授权访问 | 不携带 Token 调用接口 | 全部返回 401 |
| 跨租户访问 | 用租户 A 的 Token 访问租户 B 的数据 | 全部返回 403 或空结果 |
| SQL 注入 | 在参数中注入 SQL 片段 | 无异常泄露，参数化查询生效 |
| 越权操作 | 低权限角色执行高权限操作 | 全部返回 403 |
| 敏感字段泄露 | 检查接口返回是否包含密码、Token 等 | 无敏感明文字段 |
| 依赖漏洞扫描 | `mvn dependency-check` 或 `npm audit` | 无 HIGH/CRITICAL 级漏洞 |

> 检查结果记录在 `SPEC-XXX-9-验收报告.md` 的"安全验收"节。
